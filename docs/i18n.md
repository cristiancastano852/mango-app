# Internacionalización (i18n)

## Stack

- **vue-i18n v11** para traducciones en el frontend (Vue)
- **Laravel native translations** (`__()`) para el backend
- Locale por defecto: `es` (español), fallback: `en` (inglés)

## Configuración del locale

El locale se controla desde el `.env`:

```env
APP_LOCALE=es           # Idioma principal
APP_FALLBACK_LOCALE=en  # Fallback
```

Laravel comparte el locale activo al frontend via `HandleInertiaRequests.php`, y `app.ts` sincroniza vue-i18n al iniciar la aplicación.

## Estructura de archivos

```
lang/
├── en/
│   └── messages.php       # Flash messages en inglés
└── es/
    └── messages.php       # Flash messages en español

resources/js/
├── locales/
│   ├── en.json            # Traducciones frontend inglés
│   └── es.json            # Traducciones frontend español
└── plugins/
    └── i18n.ts            # Configuración de vue-i18n
```

## Uso en componentes Vue

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n';

const { t } = useI18n();
</script>

<template>
  <h1>{{ t('employees.title') }}</h1>
  <Button>{{ t('common.save') }}</Button>
</template>
```

## Uso en PHP (controllers, etc.)

```php
return redirect()->back()->with('success', __('messages.employee_created'));
```

## Estructura de las traducciones JSON

Las claves están organizadas por sección:

| Sección              | Descripción                              |
|----------------------|------------------------------------------|
| `common`             | Palabras reutilizables (save, cancel...) |
| `nav`                | Items del sidebar/navegación             |
| `auth.*`             | Páginas de autenticación                 |
| `profile`            | Configuración de perfil                  |
| `password_settings`  | Configuración de contraseña              |
| `two_factor_settings`| Configuración 2FA                        |
| `delete_account`     | Diálogo de eliminar cuenta               |
| `appearance`         | Configuración de apariencia              |
| `settings_layout`    | Layout y navegación de settings          |
| `employees`          | CRUD de empleados                        |
| `time_clock`         | Página de fichaje                        |

## Agregar nuevas traducciones

1. Añadir la clave en `resources/js/locales/es.json`
2. Añadir la misma clave en `resources/js/locales/en.json`
3. Usar `t('seccion.clave')` en el componente

## Agregar un nuevo idioma

1. Crear el archivo `resources/js/locales/{locale}.json` con todas las claves
2. Añadir el idioma en `resources/js/plugins/i18n.ts`:
   ```ts
   import fr from '@/locales/fr.json';
   // ...
   messages: { es, en, fr }
   ```
3. Crear `lang/{locale}/messages.php` para los flash messages del backend
4. Actualizar el tipo `'es' | 'en'` en el plugin para incluir el nuevo locale
